智斧 4.0 制作教程：从界面原型到 Windows 可执行程序

写在前面：这篇嘢记录智斧 4.0 嘅制作全过程——一个面向 Windows 嘅桌面软件管理工具，从无到有，全部由 AI 协作搞掂。呢次经历让我确信：唔识写代码嘅人，一样可以用 AI 做出真正用得嘅 Windows 软件。

智斧系咩来嘅

智斧 4.0 系一个轻量级 Windows 桌面工具，用来看自己台电脑装咗咩软件，提供卸载、搜索、导出、打开安装目录，同埋残留清理辅助功能。

佢唔系一个网页，唔系一个 script——系一个真正可以双击运行嘅 .exe 程序，用经典 Windows 工具风格界面，内置完整嘅软件管理功能。

一句话引发嘅开发

同之前卸载 360 嘅经历几似，今次开发智斧都唔系因为我谂住写代码——而系我发现市面上好缺少一个干净、轻量嘅软件管理工具。啲所谓嘅「卸载工具」自己就系流氓软件。

我打开 Claude，讲咗一句话：

"帮我做一个可以查看已安装软件、可以卸载、可以清理残留嘅 Windows 工具。"

之后嘅所有嘢——界面设计、后端开发、功能实现、图标制作、EXE 打包——全部由 AI 协作完成。我主要负责表达需求同检查结果。

开发方式：自然语言驱动

整个项目嘅开发流程几特别下：

  • 我用中文描述想要嘅软件方向同功能
  • AI 理解需求，编写同修改代码
  • 我透过截图反馈界面问题
  • AI 调整布局、颜色、菜单
  • AI 自己打包成 EXE、启动验证
  • 我继续反馈，AI 继续迭代

呢种模式大幅降低咗桌面软件开发嘅门槛。我唔使操作复杂嘅工程细节，唔使手动处理打包、图标、依赖同验证流程，只要持续表达目标、选择方案同检查结果就得啦。

技术选型

智斧采用前后端结合嘅轻量桌面方案：

  • 前端：HTML + CSS + JavaScript
  • 桌面壳：pywebview
  • 后端：Python
  • 数据来源：Windows 注册表、AppX 包信息、文件系统
  • 打包工具：PyInstaller
  • 图标处理：Pillow

呢个方案嘅优势系开发速度快，界面易调整，Python 后端可以直接调用 Windows 本地能力，最后用 PyInstaller 打包成单文件 EXE。

核心功能

1. 读取软件列表

从 Windows 注册表三个位置读取已安装软件资讯：

  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
  • HKLM\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall
  • HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

读取字段包括名称、版本、发布者、安装日期、大小、位置、卸载命令等。读取完成后过滤系统组件、补丁更新同重复条目，按名称排序。

2. 读取 Microsoft Store Apps

透过 PowerShell 嘅 Get-AppxPackage 读取 Microsoft Store 安装嘅应用，用户可以在菜单切换查看「桌面软件」同「Store 应用」。

3. 软件操作能力

  • 卸载：执行软件嘅卸载命令
  • 打开安装目录
  • 复制卸载命令到剪贴板
  • 导出 HTML：将软件列表导出为文件
  • 搜索软件名称
  • 注册表条目定位
  • 删除注册表条目

4. 卸载后残留扫描

执行卸载之后，自动扫描以下残留项：

  • 软件安装目录
  • 桌面快捷方式
  • 开始菜单快捷方式
  • 卸载注册表项
  • 软件相关注册表路径

扫描结果展示畀用户确认之后，再执行清理。同时做咗严格嘅安全边界判断，唔允许删除系统根目录、Windows 目录等高风险路径。

界面设计

界面采用深色背景、浅色文字、斑马纹列表行同高亮选中行嘅经典工具风格：

  • 顶部菜单栏：文件、操作、查看、帮助
  • 软件表格：程序名称、大小、安装时间
  • 右键菜单：卸载、强制删除、注册表条目、安装文件夹、搜索等
  • 底部搜索框：快速筛选软件
  • 底部状态栏：软件数量同占用空间

图标设计

智斧图标嘅定位系：

  • 斧头——表示清理、整理、处理问题
  • 蓝色——表示工具、系统、稳定同科技感
  • 深色底——同软件界面风格统一

AI 生成咗几个方案，我拣咗方案一，AI 嵌入 EXE 并处理埋 Windows 图标缓存问题。

打包与体积优化

AI 用 PyInstaller 打包，主要参数：

  • --onefile：打包成单个 EXE
  • --windowed：唔显示控制台窗口
  • --icon：指定 EXE 图标
  • --exclude-module：排除唔需要嘅大型依赖

初始打包体积几大下，排除咗无用依赖之后，最终控制在大约 25 MB。

安全边界

开发呢类系统工具嘅时候，AI 一直留意安全：

  • 唔会自动删除用户文件
  • 唔会静默删除注册表
  • 清理残留之前一定要弹出确认
  • 唔允许删除系统关键目录
  • 替换 EXE 之前净系结束目标程序进程

总结：自然语言开发嘅启示

智斧 4.0 嘅制作过程可以概括为：

界面设计 → Windows 数据读取 → 卸载操作 → 残留扫描 → 菜单交互 → 图标设计 → EXE 打包 → 体积优化 → 启动验证

呢个项目让我见到一种新嘅软件生产方式：

自然语言需求 → AI 分析工程 → AI 编写代码 → AI 打包验证 → 用户反馈 → AI 继续迭代

用户主要负责表达目标同判断结果，AI 负责将目标转化为工程实现。呢种模式适合个人学习、产品验证同小型工具开发。

我相信，未来会有越来越多咁样嘅软件——唔系由专业程序员一行行敲出来嘅，而系由一个普通同 AI 倾出来嘅。
