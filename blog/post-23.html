智斧 4.0 制作教程：从界面原型到 Windows 可执行程序

写在前面：本文记录智斧 4.0 的制作全过程——一个面向 Windows 的桌面软件管理工具，从无到有，全部由 AI 协作完成。这段经历让我确信：不懂代码的人，也能用 AI 做出真正可用的 Windows 软件。

什么是智斧

智斧 4.0 是一个轻量级 Windows 桌面工具，用于查看本机已安装软件，并提供卸载、搜索、导出、打开安装目录、注册表相关入口和残留清理辅助功能。

它不是一个网页，不是一个脚本——是一个真正可双击运行的 .exe 程序，采用经典 Windows 工具风格界面，内置完整的软件管理功能。

一句话引发的开发

和之前卸载 360 的经历很像，这次开发智斧也不是因为我想写代码——而是我发现市面上缺少一个干净、轻量的软件管理工具。那些所谓的「卸载工具」自己就是流氓软件。

我打开 Claude，说了一句话：

"帮我做一个能查看已安装软件、能卸载、能清理残留的 Windows 工具。"

接下来的所有事情——界面设计、后端开发、功能实现、图标制作、EXE 打包——全部由 AI 协作完成。我主要负责表达需求和检查结果。

开发方式：自然语言驱动

整个项目的开发流程非常特别：

  • 我用中文描述想要的软件方向和功能
  • AI 理解需求，编写和修改代码
  • 我通过截图反馈界面问题
  • AI 调整布局、颜色、菜单
  • AI 自己打包成 EXE、启动验证
  • 我继续反馈，AI 继续迭代

这种模式大幅降低了桌面软件开发门槛。我不需要操作复杂的工程细节，不需要手动处理打包、图标、依赖和验证流程，只需要持续表达目标、选择方案和检查结果。

技术选型

智斧采用前后端结合的轻量桌面方案：

  • 前端：HTML + CSS + JavaScript
  • 桌面壳：pywebview
  • 后端：Python
  • 数据来源：Windows 注册表、AppX 包信息、文件系统
  • 打包工具：PyInstaller
  • 图标处理：Pillow

这个方案的优势是开发速度快，界面易调整，Python 后端可以直接调用 Windows 本地能力，最终用 PyInstaller 打包成单文件 EXE。

核心功能

1. 读取软件列表

从 Windows 注册表三个位置读取已安装软件信息：

  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall
  • HKLM\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Uninstall
  • HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Uninstall

读取字段包括名称、版本、发布者、安装日期、大小、位置、卸载命令等。读取完成后过滤系统组件、补丁更新和重复条目，按名称排序。

2. 读取 Microsoft Store Apps

通过 PowerShell 的 Get-AppxPackage 读取 Microsoft Store 安装的应用，用户可以在菜单中切换查看「桌面软件」和「Store 应用」。

3. 软件操作能力

  • 卸载：执行软件的卸载命令
  • 打开安装目录
  • 复制卸载命令到剪贴板
  • 导出 HTML：把软件列表导出为文件
  • 搜索软件名称
  • 注册表条目定位
  • 删除注册表条目

4. 卸载后残留扫描

执行卸载后，自动扫描以下残留项：

  • 软件安装目录
  • 桌面快捷方式
  • 开始菜单快捷方式
  • 卸载注册表项
  • 软件相关注册表路径

扫描结果展示给用户确认后，再执行清理。同时做了严格的安全边界判断，不允许删除系统根目录、Windows 目录等高风险路径。

界面设计

界面采用深色背景、浅色文字、斑马纹列表行和高亮选中行的经典工具风格：

  • 顶部菜单栏：文件、操作、查看、帮助
  • 软件表格：程序名称、大小、安装时间
  • 右键菜单：卸载、强制删除、注册表条目、安装文件夹、搜索等
  • 底部搜索框：快速筛选软件
  • 底部状态栏：软件数量和占用空间

图标设计

智斧的图标定位是：

  • 斧头——表示清理、整理、处理问题
  • 蓝色——表示工具、系统、稳定和科技感
  • 深色底——与软件界面风格统一

AI 生成了多个方案，我选择方案一，AI 嵌入 EXE 并处理了 Windows 图标缓存问题。

打包与体积优化

AI 使用 PyInstaller 打包，主要参数：

  • --onefile：打包成单个 EXE
  • --windowed：不显示控制台窗口
  • --icon：指定 EXE 图标
  • --exclude-module：排除不需要的大型依赖

初始打包体积较大，经过排除无用依赖后，最终控制在约 25 MB。

安全边界

开发这类系统工具时，AI 始终注意安全：

  • 不会自动删除用户文件
  • 不会静默删除注册表
  • 清理残留前必须弹出确认
  • 不允许删除系统关键目录
  • 替换 EXE 前只结束目标程序进程

总结：自然语言开发的启示

智斧 4.0 的制作过程可以概括为：

界面设计 → Windows 数据读取 → 卸载操作 → 残留扫描 → 菜单交互 → 图标设计 → EXE 打包 → 体积优化 → 启动验证

这个项目让我看到了一种新的软件生产方式：

自然语言需求 → AI 分析工程 → AI 编写代码 → AI 打包验证 → 用户反馈 → AI 继续迭代

用户主要负责表达目标和判断结果，AI 负责把目标转化为工程实现。这种模式适合个人学习、产品验证和小型工具开发。

我相信，未来会有越来越多这样的软件——不是由专业程序员一行行写出来的，而是由一个普通人和 AI 聊出来的。
