零成本搭建 AI 内容网站全记录

用 AI 生成网站、GitHub 管理代码、Vercel 自动部署——打造一个纯静态内容站点的完整过程。

为什么做这个网站

AI 技术发展太快了，每天都有新工具、新模型、新应用出现。我想做一个中文的 AI 资源站，把学习笔记、工具推荐、实验项目整理起来，帮自己和更多人跟上这个节奏。

目标很简单：零成本、快速上线、容易维护。

技术选型

第一版没有用复杂的框架，纯静态站点 + CDN 部署就够了：

  • 前端： 纯 HTML + CSS + JavaScript
  • 代码管理： GitHub
  • 部署： Vercel（免费版）
  • 域名： 先用 xxx.vercel.app，后续换自定义域名

为什么不用框架？第一版核心是验证内容和需求，HTML/CSS/JS 零依赖、零构建，改了就生效。等后续内容多了再升级到 Astro 或 Next.js 也不迟。

项目结构

ai-site/
├── index.html      # 首页
├── blog.html       # 博客列表
├── tools.html      # 工具导航
├── about.html      # 关于页
├── blog/           # 博文
├── css/style.css   # 全局样式
└── js/main.js      # 交互逻辑

结构清晰，每个页面职责明确。CSS 和 JS 是全局共享的，新增页面时只需要引用它们。

设计风格

没有用 UI 组件库，所有样式自己写 CSS 变量，好处是轻量和灵活：

  • 紫蓝渐变作为品牌色（--color-primary: #6366f1）
  • 响应式布局，移动端适配
  • 卡片式 UI，内容清晰易读
  • 粘性导航栏 + 毛玻璃效果
  • 暗色/亮色兼容（纯白底色 + 柔和阴影）

工作流：从编辑到上线

整个流程只有三步，全程自动化：

第一步：编辑内容

网站的博客文章和工具数据都定义在 js/main.js 中，想加一篇新博文只需要在 blogPosts 数组里添加一条记录：

const blogPosts = [
  {
    id: 7,
    title: '新文章标题',
    tag: '学习',
    excerpt: '文章简介...',
    date: '2026-05-21',
    readTime: '5 分钟',
  },
  // ...
];

同时可以在 blog/ 目录下新建一个 HTML 文件作为完整博文。（早期内容不多时，也可以直接用 excerpt 字段做摘要展示，点击再展开。）

第二步：提交到 GitHub

编辑器改完后，终端跑三条命令：

cd ~/projects/ai-site
git add .
git commit -m "添加新博文：xxx"
git push

git add . 暂存所有改动，git commit 打上快照，git push 推送到 GitHub。

第三步：Vercel 自动部署

只要 GitHub 仓库连接了 Vercel，每次 git push 都会自动触发重新部署。Vercel 会拉取最新代码，部署到全球 CDN，一般 1-2 分钟生效。你什么都不用做，刷新网页就能看到更新。

成本明细

这可能是最让我意外的地方——几乎零成本：

  • GitHub： 免费（公开仓库）
  • Vercel： 免费（每月 100GB 带宽，个人站完全够用）
  • 域名： 先用 xxx.vercel.app，后续换 .com 约 70 元/年
  • AI 辅助： Claude Code 生成网站结构和内容

如果有自定义域名的需求，一年总成本不到 100 块钱。

反爬措施

内容站免不了被采集，早期可以做一些低成本防护：

  • 接入 Cloudflare 免费版，开启 Bot Fight Mode
  • 关键内容用 JavaScript 动态加载，增加采集成本
  • 定期查看 Vercel Analytics 的流量来源

对于个人站来说，做到让采集者需要专门写脚本就够了——不值得为你的内容投入这个精力，就是最好的防护。

未来计划

  • 1 周： 网站上线，基础内容填充
  • 1 个月： 持续更新，每天短内容 + 每周深度文章
  • 3 个月： 做 SEO，积累自然搜索流量

技术上的升级计划：

  • 从纯 HTML 迁移到 Astro（组件化、Markdown 写作）
  • 接入邮件订阅（Mailchimp 等）
  • 添加搜索功能

总结

这个流程最让我喜欢的地方是：写内容 -> 写代码 -> 推送 -> 上线，整个链路不超过 5 分钟就能完成一次更新。没有服务器需要维护，没有数据库需要操心，没有构建部署的等待时间。

对于想做个人内容网站的朋友，这是一个性价比极高的起点。等流量起来了，内容积累够了，再逐步升级也不晚。

最重要的不是技术栈，是 开始写。
